> 项目使用java的开发者一定会为新配环境变量而头大，sdkman很好的解决了系统sdk管理的痛点，仅需简单的几行命令就可以完成sdk的安装，更改默认版本。再也不用担心环境变量的问题。

### 安装

既然是命令行工具，就只能用命令行方式来安装(linux，mac直接用shell，windows用powerShell)
第一步、下载

```bash
curl -s "https://get.sdkman.io" | bash
```

第二步、执行脚本

```bash
source "$HOME/.sdkman/bin/sdkman-init.sh"
```

第三步、检查是否安装成功

```bash
sdk version
```

输出版本信息即安装成功

<!--more-->

### 使用

1. 输入`sdk list`可以查看有那些sdk可以安装，系统已经安装那些sdk

2. 安装jdk（其他sdk安装类似）

```bash
sdk install java
```

接下来会看到

```bash
Downloading: java 8u111
In progress...
######################################################################## 100.0%
Installing: java 8u111
Done installing!
```

接下来会询问是否作为默认版本

3. 安装特定版本sdk
   列出可安装版本,如果本地已经安装也会看到标记

```bash
sdk list java
```

安装8u131版本jdk

```bash
sdk install java 8u131
```

```bash
sdk install maven
```

4. 移除SDK

```bash
sdk uninstall java
# 或者
sdk rm java
```

5. 查看当前安装的sdk版本

```bash
sdk current java
```
